# TestTask

## Тестовое задание: Компонент календаря (Vue 3)

### Задача
Реализовать компонент календаря на Vue.js (версии 2 и выше; в проекте используется Vue 3):
- Переключение месяцев (влево/вправо).
- Выбор конкретной даты кликом по дню.
- На клик по дню эмитить событие с выбранной датой.
- При инициализации компонент принимает дату в формате `YYYY-MM-DD` и показывает соответствующие месяц и день. Если дата не передана — берётся текущий день.
- Реализовать смену языка (названия месяцев и дней недели).

### Стек
- Vue 3 + Vite
- JavaScript / Composition API
- Intl.DateTimeFormat для локализации

### Структура
- `test-calendar/src/components/Calendar.vue` — компонент календаря
- `test-calendar/src/App.vue` — демо-страница с выбором языка и начальной даты

### Быстрый старт
```bash
cd test-calendar
npm i
npm run dev
```
Откройте адрес из консоли (обычно `http://localhost:5173`).

### Использование компонента
Пример (см. `src/App.vue`):
```vue
<template>
  <Calendar :initial-date="initial || undefined"
            :locale="locale"
            @day-click="onDayClick" />
  <p>Выбрано: {{ selected }}</p>
  
</template>

<script setup>
import { ref } from 'vue'
import Calendar from './components/Calendar.vue'

const locale = ref('ru-RU')        // язык
const initial = ref('2025-11-05')  // начальная дата (опционально)
const selected = ref('')

function onDayClick(dateIso) {
  selected.value = dateIso // YYYY-MM-DD
}
</script>
```

#### Пропсы
- `initial-date?: string` — начальная дата в формате `YYYY-MM-DD`. Если не задана или невалидна — используется сегодняшний день.
- `locale?: string` — локаль для названий месяцев и дней недели (например, `ru-RU`, `en-US`, `de-DE`).

#### События
- `day-click` — эмитится при выборе дня. Пэйлоад: строка в формате `YYYY-MM-DD`.

### Локализация
- Названия месяцев и дней недели формируются через `Intl.DateTimeFormat(locale, ...)`.
- Неделя начинается с воскресенья (как в референс‑макете).

### Валидация даты
- Формат проверяется регулярным выражением `^([0-9]{4})-([0-9]{2})-([0-9]{2})$` и дополнительной проверкой реальности даты через `new Date(y, m-1, d)`.

### Стили и внешний вид
- Плоская рамка контейнера, компактные ячейки дней, выбранный день с синей обводкой.
- Все дни строго выравниваются под заголовками дней недели; элементы не выходят за рамки контейнера.
